SOQL Guide For JavaScript And Python Developers

A project-local guide to Salesforce queries using the personnel tracker sample.

Mental Model

SOQL reads Salesforce records from objects like Personnel__c and Personnel_Assignment__c. It looks familiar if you know SQL, but it follows Salesforce metadata, relationships, sharing, and governor limits.

If you know...Think of SOQL as...Important difference
SQLSELECT over Salesforce objectsNo arbitrary joins and no normal SELECT *.
TypeScript arraysA server-side filter before Apex or LWC sees dataYou must select every field you plan to read.
Python ORMsA query layer over recordsGovernor limits reward querying collections once.

Where SOQL Belongs

In this repo, selectors own SOQL, services own business behavior, and scripts or LWCs call services.

LWC or script
  -> Apex controller or service
    -> selector
      -> SOQL
Open PersonnelSelector.cls and PersonnelService.cls side by side. The selector describes database reads; the service describes intent.

Basic Query Shape

List<Personnel__c> engineers = [
    SELECT Id, Employee_Id__c, Display_Name__c, Department__c, Role__c
    FROM Personnel__c
    WHERE Department__c = 'Engineering'
        AND Status__c = 'Active'
    ORDER BY Last_Name__c, First_Name__c
    LIMIT 50
];

Custom objects and fields end in __c. Always list the fields you plan to read and add ORDER BY when row order matters.

Bind Variables

String department = 'Engineering';
Set<String> statuses = new Set<String>{ 'Active', 'On Leave' };

List<Personnel__c> people = [
    SELECT Id, Employee_Id__c, Display_Name__c, Status__c
    FROM Personnel__c
    WHERE Department__c = :department
        AND Status__c IN :statuses
];

Bind variables are the Apex version of parameterized query values. Prefer them over dynamic string concatenation.

Relationship Queries

Parent To Child

SELECT Employee_Id__c, Display_Name__c,
    (SELECT Assignment_Name__c, Status__c FROM Assignments__r)
FROM Personnel__c
WHERE Employee_Id__c LIKE 'SAMPLE-%'

Child To Parent

SELECT Assignment_Name__c, Status__c, Personnel__r.Display_Name__c
FROM Personnel_Assignment__c
WHERE Status__c = 'Active'

Aggregate Queries

for (AggregateResult row : [
    SELECT Department__c department, COUNT(Id) activePersonnel
    FROM Personnel__c
    WHERE Status__c = 'Active'
    GROUP BY Department__c
]) {
    System.debug(row.get('department') + ': ' + row.get('activePersonnel'));
}

Bulk-Safe Pattern

Gather keys, query once, and use a map. Do not query once per record inside a loop.

Set<String> employeeIds = new Set<String>();
for (Personnel__c person : incomingPeople) {
    employeeIds.add(person.Employee_Id__c);
}

Map<String, Personnel__c> existingByEmployeeId = new Map<String, Personnel__c>();
for (Personnel__c existing : PersonnelSelector.selectByEmployeeIds(employeeIds)) {
    existingByEmployeeId.put(existing.Employee_Id__c, existing);
}

Security Habits

ConcernWhat it controlsHabit
SharingWhich records the user can seeUse with sharing unless you have a deliberate reason not to.
CRUD/FLSWhich objects and fields the user can accessLearn user-mode operations and stripInaccessible before building real UI controllers.
