Personnel Tracker Apex Walkthrough

A guided tour for TypeScript and Python developers learning Salesforce Apex.

How To Use This Guide

Work through the project in order: deploy metadata, run anonymous Apex, trace the service and selector classes, inspect the trigger path, run tests, run Batch Apex, then study SOQL as its own skill.

Mental Model

Setup

sf org login web --alias PersonnelDev --set-default
sf project deploy start --source-dir force-app
sf org assign permset --name Personnel_Sample_Admin
sf org open
Checkpoint: in Object Manager, confirm the Personnel and Personnel Assignment objects exist.

1. Metadata Is Source Code

Start with Personnel__c.object-meta.xml, Employee_Id__c.field-meta.xml, and Termination_Date_Required.validationRule-meta.xml. Notice that the object, fields, and validation rules are all stored as deployable source files.

Exercise: add a picklist value to Department__c, deploy, and inspect the field in Object Manager.

2. Anonymous Apex As A REPL

sf apex run --file scripts/apex/01_insert_sample_personnel.apex
01_insert_sample_personnel.apex
  -> PersonnelService.upsertPersonnel(...)
    -> PersonnelService.normalizeForSave(...)
    -> PersonnelSelector.selectByEmployeeIds(...)
    -> Database.upsert(...)
      -> PersonnelTrigger
        -> PersonnelTriggerHandler

The important point is that the script creates records, but the service owns the save behavior. The trigger still runs because DML happened.

3. SOQL Belongs In Selectors

sf apex run --file scripts/apex/02_query_headcount.apex
sf apex run --file scripts/apex/06_soql_practice.apex

Read docs/soql-guide.md and PersonnelSelector.cls. The selector hides query details so service methods can read like business operations. The SOQL practice script demonstrates binds, relationship queries, aggregate queries, and SOQL for-loops.

4. Services Own Business Rules

sf apex run --file scripts/apex/03_create_assignments.apex

Read PersonnelAssignmentService.cls. It rejects inactive personnel, inserts department assignments in bulk, and closes open work when personnel are terminated.

5. One Thin Trigger

sf apex run test --class-names PersonnelTriggerHandlerTest --result-format human --wait 10

The trigger delegates immediately to PersonnelTriggerHandler.cls. This keeps trigger logic testable and easier to reason about.

6. Bulk Safety

sf apex run test --class-names PersonnelServiceTest --result-format human --wait 10

Find upsertPersonnel_handlesTwoHundredRecords. The test exists because Salesforce can pass many records through one transaction. Look for Set, Map, one SOQL query, and one DML call.

7. Batch Apex

sf apex run --file scripts/apex/05_run_review_batch.apex
sf apex run test --class-names PersonnelReviewBatchTest --result-format human --wait 10

Read PersonnelReviewBatch.cls. Batch Apex separates finding records, processing chunks, and finishing the job.

8. Guided Change

Add a rule that assignments cannot exceed 40 hours per week. Add the failing test first in PersonnelAssignmentServiceTest.cls, then update PersonnelAssignmentService.cls, then re-run the test.

9. SOQL As A Skill

Open docs/soql-guide.html in a browser or read docs/soql-guide.md. Then add a selector method that finds personnel by clearance level and call it from a new anonymous Apex script.

Cleanup

sf apex run --file scripts/apex/99_cleanup_sample_data.apex

The cleanup script deletes assignments before personnel and only targets sample IDs.
