Lightning UI Walkthrough

A Salesforce Lightning Web Component example for the personnel tracker.

What This Adds

The UI example lives in force-app/main/default/lwc/personnelDashboard and is powered by PersonnelDashboardController.cls.

FilePurpose
personnelDashboard.htmlTemplate for filters, metrics, tables, and assignment form.
personnelDashboard.jsClient-side state, wired reads, imperative writes, and toast events.
personnelDashboard.cssComponent-scoped styling.
personnelDashboard.js-meta.xmlLightning App Builder exposure and builder property.
PersonnelDashboardController.clsUI controller that returns LWC-friendly wrapper data.

Deploy

sf project deploy start --source-dir force-app
sf org assign permset --name Personnel_Sample_Admin
sf apex run --file scripts/apex/01_insert_sample_personnel.apex
sf apex run --file scripts/apex/03_create_assignments.apex

Add It In Lightning App Builder

  1. In Setup, open Lightning App Builder.
  2. Create a new App Page named Personnel Dashboard.
  3. Drag the custom component Personnel Dashboard onto the page.
  4. Save and activate the page.

Read Path

@wire(getDashboardData, { departmentFilter: '$selectedDepartment' })
  -> PersonnelDashboardController.getDashboardData(...)
    -> PersonnelService.findActiveByDepartment(...)
    -> PersonnelSelector.selectByDepartment(...)
    -> SOQL
Use @wire for cacheable reads. The dashboard reloads when selectedDepartment changes.

Write Path

handleCreateAssignment()
  -> createAssignment({ request: { ... } })
    -> PersonnelDashboardController.createAssignment(...)
      -> PersonnelAssignmentService.createAssignment(...)
      -> DML insert
Use imperative Apex for writes because the user controls exactly when the action happens.

Lightning Concepts

  • The LWC bundle files share the same name as the component folder.
  • The component uses Lightning base components instead of hand-built form controls.
  • The .js-meta.xml file exposes the component to App, Home, and Record pages.
  • The @api defaultDepartment field can be configured in Lightning App Builder.
  • After a successful write, refreshApex reloads wired data.
  • Modern LWC development expects explicit component API versioning; Salesforce docs note versioning support beginning in Winter '24 and required versioning beginning in Spring '25.
